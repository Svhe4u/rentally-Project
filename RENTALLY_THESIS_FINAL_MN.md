# RENTALLY - ДИПЛОМЫН АЖЛЫН СУДАЛГАА

## НЭГ. СЭДВИЙН СУДЛАГДСАН БАЙДАЛ / СУДАЛГААНЫ ОНОЛ АРГА ЗҮЙ

### 1.1 Ерөнхий судалгаа

Монголын үл хөдлөх хөрөнгөний түрээсийн салбар сүүлийн жилүүдэд хүчтэй хөгжиж байгаа нь түүний цахим үйлчилгээнээс шалтгаалж байна. Дэлхий даяар Airbnb, Booking.com, Zillow зэрэг олон улсын платформууд 2008 оноос хойш үл хөдлөх хөрөнгөний түрээсийг онлайнаар хангуулж байна. Мобайл программуудын ашигласу 78%-д хүрэв (2023). Гэтэл өнөөгийн байдлаар Монголын зах зээлд өргөнөөр ашигласан нэгдмэл платформ байхгүй байна. Үл хөдлөх хөрөнгө-түрээслүүлэгчдийн хоорондын үнэлгээ системийн дутагдал, төлбөрийн аюулгүй байдлын дутагдал, мессежийн систем байхгүй байна. Rentally систем нь энэ асуудлыг шийдвэрлэхээр хөгжүүлэгдэж байгаа цахим платформ юм.

### 1.2 Одоогийн системийн судалгаа

Rentally платформ 2023 онд Монголын үл хөдлөх хөрөнгөний түрээсийн салбарын дижиталжуулалтын зорилгоор хөгжүүлэгдэж эхэлсэн.

**Системийн цар хүрээ ба хүрээ:**
- Хэрэглэгчдийн төрөл: Ердийн хэрэглэгч (renter), Зуучлагч (broker), Админ
- Үйл ажиллагааны салбарууд: Үл хөдлөх хөрөнгөний жагсаалт, түрээсийн захиалга, үнэлгээ, мессеж, төлбөр
- Ашигласын хүрээ: Нийслэл Улаанбаатар ба түүний дүүргүүд

## НЭГ. ХИЙГДЭХ СИСТЕМИЙН СУДАЛГАА

### 1.3.1 Scenario (Бодит үйл явдлыг зохиол байдлаар дүрслэлэх)

**Сценарий 1: Түрээслэгч үл хөдлөх хөрөнгөйг хайж, захиалга үүсгэх**

Түрээслэгч байшил түрээслүүлэхээр шийдсэн гэж үзье. Тэр хэрэглэгч Rentally мобайл аппликейшнд нэвтрэн орж, өндөрлөж байх хүсэлтээ (Улаанбаатар, Баянзүрх дүүрэг) сонгоно. Систем тухайн бүс нутагт давхцаж буй бүх үл хөдлөх хөрөнгөний мэдээллийг (зургууд, үнэ, үнэлгээ) харагдах байдлаар нэг дор харуулна. Түрээслэгч түүнээс өөрт тохирох үл хөдлөх хөрөнгөйг сонгон, үл хөдлөх хөрөнгөний дэлгэрүүлэлтийг үзнэ (өрөөний тоо, ванны тоо, байрлал гэх мэт). Дараа нь түрээслэгч захиалгын хуудсыг нээж, өдөр тогтоно (эхлэх ба төгсөх өдөр). Систем төлбөрийн хэмжээг автоматаар тооцоолж (өдөрүүдийн тоо × үнэ) харуулна. Түрээслэгч захиалгыг "pending" төлөвтөө хадгалдаг. Дараа нь төлбөрийн хэсэг рүү шилжиж, төлбөрийн мэдээлэл оруулж баталгаажуулна. Төлбөр амжилттай баталгаажмагц захиалга "confirmed" төлөвт шилжиж, зуучлагч болон түрээслэгч аль алиа мэдэгдэл авдаг. Зуучлагч захиалгыг хүлээн зөвшөөрвөл, түрээслэгч захиалгын мэдээлэл, зуучлагчийн холбоо авах хаяг, мессежийн сүлжээ авдаг.

**Сценарий 2: Зуучлагч үл хөдлөх хөрөнгөйг нэмэх ба удирдах**

Зуучлагч "broker" ролийг сонгон Rentally системд бүртгүүлнэ. Систем зуучлагчийн BrokerProfile объект үүсгэнэ (статус: "pending"). Админ зуучлагчийн лицензийн баримтыг үзэн баталгаажуулахад (статус: "approved") алдартай болно. Зуучлагч POST /api/listings/ руу үл хөдлөх хөрөнгөний ерөнхий мэдээлэл (нэр, хаяг, үнэ) оруулнэ. Зургуудыг ачаалдаг (дөрвөн ба түүнээс дээш зураг). Үл хөдлөх хөрөнгөний дэлгэрүүлэлт (өрөөний тоо, ванны тоо, хэмжээ, байнгын зардал үнэлгээ) оруулдаг. Үл хөдлөх хөрөнгөний шинж чанарууд (WiFi, паркинг гэх мэт) нэмдэг. Үл хөдлөх хөрөнгө активтай болж, түрээслэгчдээ харагдаж эхлэнэ.

### 1.3.2 Use case (Ашигласын аргууд)

**Use Case 1: Энгийн хэрэглэгч (Renter) баялагыг хайх, захиалга үүсгэх**
1. Хэрэглэгч мобайл аппликейшнд нэвтрэх
2. Бүс нутаг (region), төрөл (category), үнийн хүрээ сонгоно
3. Системээс баялагын жагсаалтыг авна
4. Өөрт тохирох баялагыг сонгоно
5. Баялагын дэлгэрүүлэлтийг үзнэ (зургууд, үнэлгээ, байлалчийн мэдээлэл)
6. Захиалгыг үүсгэнэ (эхлэх, төгсөх өдөр)
7. Төлбөр гүйцэтгэнэ
8. Захиалга "confirmed" болно
9. Мессежээр байлалчтай холбоо авна

**Use Case 2: Байлалч (Broker) баялагыг удирдах**
1. Байлалч системд нэвтрэх ("broker" ролийн хэрэглэгч)
2. Компанийн мэдээлэл оруулна (нэр, регистрийн дугаар, лицензи)
3. Админ баталгаажуулахыг хүлээнэ
4. Баялага нэмнэ (мэдээлэл, зургууд, дэлгэрүүлэлт)
5. Баялагыг идэвхжүүлнэ
6. Захиалгуудыг батлах/үгүйсгэх
7. Түрээслэгчдийн үнэлгээ, сэтгэгдлийг үзнэ

**Use Case 3: Админ (Administrator) систем удирдах**
1. Админ системд нэвтрэх
2. Байлалчдыг баталгаажуулах (баримтын үзүүлэлт)
3. Баялагуудыг модерац хийх (зөв биш агуулгаа шүүрэх)
4. Хэрэглэгчдийн эрхийг удирдах
5. Төлбөрийн гүйлгээний журнал хянах

### 1.4 Архитектурын сонголт

**Программ хангамжийн архитектур:**

Rentally системийг хөгжүүлэхэд Django REST Framework ашигласан. Django нь Python хэл дээр ажилладаг, MVC concept-т суурилсан, веб системүүдийн хөгжүүлэлтийн хурд болон аюулгүй байдлыг сайжруулдаг framework юм. Django нь ORM ашиглан өгөгдлийн сангийн query-г хялбар хийдэг тул системийг хөгжүүлэхэд тохиромжтой.

PostgreSQL өгөгдлийн сан ашигласан бөгөөд production орчны хувьд бүхэлдээ бүтцийн өгөгдөл, нарийн асуулга, индексийг дэмжиж байдаг. Django нь API болон web UI хоёуланг зэрэг үүсгэх боломжтой тул системийг Mobile болон Web зориулалтаар зэрэг ашиглах боломжтой архитектуртай болгож өгдөг.

Frontend хэсэг нь React Native + Expo ашиглан мобайл UI-г хийдэг. Хэрэглэгч мобайл аппликейшнд нэвтэрч баялагыг хайх, захиалга үүсгэх, төлбөр хийх зэрэг үйлдлүүдийг гүйцэтгэнэ. Backend нь сервер талын логик, захиалгын баталгаажуулалт, суудлын менежментийг гүйцэтгэнэ.

Cloudinary ашиглан баялагын зургуудыг CDN дээр хадгалдаг. JWT (djangorestframework-simplejwt) ашиглан хэрэглэгчдийн аюулгүй байдлыг хангаж байдаг. Django Channels ашиглан WebSocket-ын шуурхан мессежийн үйлчилгээ хүргэдэг.

**Техник хангамжийн архитектур:**

Хөгжүүлэлтийн орчны хувьд:
- CPU: 2 core процессор (Intel Core i3 түвшин)
- RAM: 4GB
- Storage: 20GB сул зайтай байх
- OS: Windows 10+ / Mac / Ubuntu 20.04+
- Python: 3.10+
- Node.js: 18.x+
- PostgreSQL: 14+

Үйлчилгээний орчны хувьд (Production):
- CPU: 4+ core
- RAM: 8-16GB
- Storage: SSD 100GB+
- OS: Linux Ubuntu server 20.04+
- Docker container deployment
- AWS EC2 / Heroku / DigitalOcean
- Nginx reverse proxy
- SSL/TLS шифрлэлт

### 1.5 Программчлалын нэмэлт судалгаа

Rentally системийг хөгжүүлэх явцад мобайл платформын хүрээнд илүү оновчтой архитектур, олон платформ дээр зэрэг ажиллуулах боломжтой технологи судлах шаардлага гарсан.

**Expo Framework (Frontend):** Expo нь React Native дээр тулгуурласан тул нэг кодын сантайгаар Android болон iOS систем дээр зэрэг ажиллах боломжийг олгодог. Mөн тоног төхөөрөмжөөс хамаарах native module-уудыг бэлэн байдлаар ашиглах давуу талтай. EAS Build ашиглан iOS, Android app-ыг хялбар build хийдэг.

**Django REST Framework (Backend):** Django нь Python хэл дээр бичигдсэн өндөр түвшний web development framework бөгөөд REST API үүсгэхэд тохиромжтой. Өгөгдлийн сангийн ORM давхарга, authentication, admin panel зэрэг угсармал боломжуудтай тул хөгжүүлэлтийн хугацаа болон архитектурын тогтвортой байдлыг нэмэгдүүлдэг.

**PostgreSQL (Database):** PostgreSQL нь хүчтэй сонголт, JSONB тип дэмжэлт, индекс дэмжиж байдаг. Transaction ACID properties хангаж байдаг тул захиалга, төлбөрийн нөхцөл хангах боломжтой.

**JWT Authentication:** Access token (15 мин validity), Refresh token (7 өдрийн validity) ашигладаг. Token blacklist ашиглан logout үед токен хүчинтэй болохгүй байдлаар хийдэг.

**Database Query Optimization:** N+1 problem-ийг шийдвэрлэхэд Django ORM-ын select_related() ба prefetch_related() ашигласан. Индекс ашиглан асуулгыг хурдасгасан.

**Cloudinary (Image Storage):** Зургийн хэмжээг автоматаар өөрчилэх, WEBP format ашиглан сүүлийн хурдны үйлчилгээ хүргэдэг. CDN ашиглан дэлхийн өнцгөөс хурдан ачаалдаг.

**WebSocket (Real-time Messaging):** Django Channels ашиглан WebSocket ашиглан шуурхан мессежийн үйлчилгээ хүргэдэг. Redis ашиглан channel layer-г хөгжүүлэх боломжтой.

**Asynchronous Task Processing:** Celery + Redis ашиглан email илгээх, зургийн обработкаа background-д хийдэг. HTTP request-ыг удаа авалцуулалт үүсэхгүйгээр асинхроноор гүйцэтгэнэ.

Expo (frontend) болон Django (backend) хоорондын өгөгдлийн солилцоо нь REST API ба JSON формат ашиглаж HTTP хүсэлт/хариу хэлбэрээр хийгддэг. Expo талдаа fetch API ашиглан Django сервер рүү хүсэлт илгээж, сервер талд хүсэлтүүдийг боловсруулж JSON өгөгдөл буцаадаг. Ингэснээр мобайл аппликейшн болон сервер хоорондын уялдаа бүрэн хангагдаж, Rentally системийн үндсэн функцийг бүрэн гүйцэтгэх боломжтой болсон.

## ДҮГНЭЛТ

Rentally систем нь Монголын үл хөдлөх хөрөнгөний түрээсийн салбарыг дижиталжуулахад чиглэгдсэн цахим платформ юм. Системийн хөгжүүлэлтийн үндсэнд REST API, JWT authentication, PostgreSQL өгөгдлийн сан, React Native frontend, Django backend гэх технологийн сонголт байна.

Энэхүү судалгааны баримталгаас дээр үндэслэнэ системийн дизайн, өгөгдлийн загвар, API дизайн, хэрэглээч интерфэйсийн дизайн гэх үе шатуудын үйл ажиллагаа явагдана.
